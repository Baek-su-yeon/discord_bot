# 알려진 한계 (Known Issues)

## 자정을 넘긴 연속 음성 세션 — 새 날 음성 누락

**증상**: X일 자정을 넘겨 X+1일까지 연속으로 음성에 접속해 있던 경우, X+1일의 음성
세션이 0개가 되어 그날은 **댓글 폴백으로 계산**됩니다.

**원인**: 자정 집계의 자정 컷(`close_open_sessions`)이 X일의 열린 세션을 자정에서 닫지만,
X+1일에는 "입장" 이벤트가 없어 새 세션이 열리지 않습니다. 이후 X+1일에 퇴장할 때
`add_session_end`는 닫을 열린 세션을 찾지 못해 무시되므로(에러·데이터 오염 없음),
X+1일의 09:00 이후 음성 체류 시간이 음성 소스로 잡히지 않고 댓글 기준으로 강등됩니다.

**영향**: 빈도 낮고 **데이터 오염은 없음**(잘못된 값이 기록되지 않으며, 댓글 폴백으로
안전하게 계산됨). 자정 직후(09:00 이전) 퇴장 케이스는 09:00~자정 클램프로 어차피
제외되는 시간대라 무해합니다.

**개선 방향(미적용)**: 자정 컷 시 같은 유저에 대해 다음 날 세션을 재오픈(`[00:00, null]`)
하면 X+1일 퇴장 이벤트가 정상적으로 닫아 음성 시간이 보존됩니다.

> 현재는 의도적으로 고치지 않은 **known limitation**입니다.
