# 디스코드 출퇴근 집계 봇

음성 채널 체류 시간과 포럼 게시물 댓글을 바탕으로 **출퇴근(공부) 시간을 집계**하고,
매일 **입퇴실·휴가 게시물을 자동 생성**하며 **집계 결과를 게시**하는 디스코드 봇입니다.
화면공유(Go Live) 미사용 시 알림도 보냅니다.

---

## 동작 개요

### 매일 자동 흐름
- **자정(00:00 KST)** — 전날(09:00~자정) 데이터를 **확정 집계**하고 결과를 보관.
  - 진행 중인 음성 세션을 자정에서 끊는 **자정 컷** 수행.
- **09:00 KST** — 게시물 생성/게시를 **이 순서**로 수행:
  1. **입퇴실 게시물** (태그 `입퇴실`) — 예: `6월 15일 입퇴실`
  2. **휴가 게시물** (태그 `휴가`) — 예: `6월 15일 휴가`
  3. **집계 결과 게시물** — 전날 자정에 확정한 결과를 게시/갱신
  - 운동 게시물은 **봇이 만들지 않습니다**(별도 자동화 담당).

> 같은 날 09:00 작업이 두 번 돌아도(봇 재시작 등) 입퇴실/휴가 게시물은 **중복 생성되지 않습니다**.

### 출석 인증 (댓글)
- 입퇴실 게시물에 **"입실"이 포함된 댓글**을 달면 그날 **출근 인정**(부분 포함 매칭).
- 자정까지 입실 댓글이 없는 참여자는 **사유 불문 자동 휴가**로 집계됩니다.

### 공부시간 계산 (음성 우선 → 댓글 폴백)
1. **1순위 — 음성 채널 체류 시간.** 해당 날짜에 유효한 음성 세션이 있으면 그것만 사용.
   - 다중 세션 합산, **09:00~자정 클램프**, 식사시간 차감 없음.
   - 종료 시각이 `null`로 남은 세션(봇이 퇴장 이벤트를 놓침)이 있으면 그날 음성 소스를 폐기하고 댓글 폴백.
2. **2순위 — 댓글 폴백.**
   - "입실"/"퇴실" 댓글을 시간순으로 페어링해 다중 세션 합산.
   - **식사 60분 차감.** 퇴실 댓글이 전혀 없으면 **가상 퇴실 18:00**으로 간주(패널티). 음수는 0.

> 화면공유 시간은 공부시간 계산에 **사용하지 않습니다**(감시 전용).

### 수상 (계산만, 게시물에는 노출 안 함)
- **개근상**: 입퇴실 게시물이 있는 모든 날에 전부 입실.
- **출근왕**: 평일 중 입실 인정일 최다.
- **공부왕**: 누적 공부시간 최다.
- **휴가왕**: 입실 안 한 날 최다(자동 휴가 포함).
- **운동왕**: 운동 게시물에 댓글 단 날 최다.
- 동률은 모두 공동 수상.

### 화면공유 알림
- 음성 입장 후(또는 화면공유 종료 후) **1시간** 동안 공유를 켜지 않으면, 해당 멤버가 있는
  **음성 채널 채팅에 @멘션**으로 알립니다.
- **09:00~18:00 사이에만** 발송하며, 켤 때까지 **1시간마다 반복**합니다.
- 타이머는 메모리(asyncio)로만 관리되어 영구 저장하지 않습니다(봇이 꺼지면 초기화).

## 아키텍처

```
fetch (수집) → aggregate (순수 계산) → render (임베드) → bot (오케스트레이션)
                       ↑                                      ↑
               sessions.json(음성 세션)              state.json(게시물 ID·보관 임베드)
```

| 파일 | 역할 |
|------|------|
| `config.py` | 환경값·상수(토큰, 채널 ID, 태그명, 키워드, 실행 시각, 파일 경로) |
| `fetch.py` | 포럼 게시물·댓글 수집(입퇴실/운동), 입실/퇴실 이벤트 분류 |
| `aggregate.py` | **순수 함수** — 공부시간(음성 우선/댓글 폴백)·자동 휴가·수상 계산 |
| `render.py` | 집계 결과 → 디스코드 임베드(오늘 출근 + 누적 시간) |
| `state.py` | `state.json`(게시물 ID), `sessions.json`(음성/화면공유 세션) 입출력 |
| `bot.py` | 봇 기동, 스케줄(자정/09:00), 음성 이벤트, 화면공유 알림, 로깅 |
| `publish_once.py` | 보조 — 09:00 게시 누락 시 집계 결과만 수동 1회 게시(아래 [보조 스크립트](#보조-스크립트--publish_oncepy-집계만-수동-게시)) |
| `tests/` | `aggregate`·`render` 단위 테스트(디스코드 비의존) |

## 설치

```bash
pip install -r requirements.txt
```

요구사항: Python 3.10+ (타입 힌트 문법), `discord.py>=2.3`, `python-dotenv`, `tzdata`.

## 설정

### 1) `.env`
`.env.example`을 복사해 `.env`를 만들고 값을 채웁니다.

```env
DISCORD_TOKEN=        # 봇 토큰 (Developer Portal)
GUILD_ID=             # 서버 ID
ATTENDANCE_CHANNEL_ID=# 출퇴근 포럼 채널 ID
SUMMARY_POST_TITLE="📊 이번 달 집계"
SUMMARY_POST_TAG="집계"   # 선택(태그 필수 포럼일 때). 입퇴실/휴가/운동과 달라야 함
```

### 2) 포럼 태그
포럼 채널에 다음 태그가 있어야 합니다(이름 정확히 일치):
- `입퇴실`, `휴가`, `운동`
- (선택) 집계 게시물용 `SUMMARY_POST_TAG` — 위 3종과 겹치면 안 됨

### 3) Discord Developer Portal — 인텐트
**Bot → Privileged Gateway Intents → MESSAGE CONTENT INTENT를 켜야 합니다.**
입퇴실 댓글의 "입실"/"퇴실" 문자열을 읽는 데 필요합니다.
봇이 100개 서버 미만이면 심사 없이 토글 가능합니다.
음성 상태·화면공유 감지는 기본 인텐트로 충분합니다.

## 실행

```bash
python bot.py
```

시작 시 한 번 집계 결과를 갱신(catch-up)한 뒤, 자정 집계·09:00 게시 스케줄을 시작합니다.

### 실행 플래그 — 기능 on/off

인자 없이 실행하면 모든 기능이 켜진 상태로 동작합니다(기존과 동일). 다음 플래그로 개별 기능을 끌 수 있습니다.

```bash
python bot.py                       # 둘 다 ON (기본)
python bot.py --no-summary          # 집계 결과 공개 게시만 OFF
python bot.py --no-stream-alert     # 라이브 미사용 알림만 OFF
python bot.py --no-summary --no-stream-alert   # 둘 다 OFF
```

| 플래그 | 효과 |
|--------|------|
| `--no-summary` | 매일 09:00에 집계 결과를 **공개 게시물에 갱신하는 동작만 생략**합니다. 입퇴실/휴가 게시물 생성과 집계 계산·보관(`state.json`의 `pending_embed`)은 그대로 유지되므로, 결과는 보관만 되고 공개되지 않습니다(필요 시 `publish_once.py`로 직접 게시 가능). |
| `--no-stream-alert` | 화면공유(라이브) **1시간 미사용 알림만 비활성화**합니다. 음성/화면공유 세션의 `sessions.json` 기록은 그대로 유지됩니다. |

> 시작 시 로그에 현재 상태를 남깁니다: `기능 상태 — 집계 결과 게시: ON/OFF, 화면공유 알림: ON/OFF`.

### 보조 스크립트 — `publish_once.py` (집계만 수동 게시)

봇이 **09:00에 꺼져 있어 게시(`run_morning_post`)가 누락된 날**, 보관된 집계 결과
(`state.json`의 `pending_embed`)를 집계 게시물에 **한 번만 반영**합니다.

```bash
python publish_once.py
```

- 로그인 → `publish_summary()` 1회 호출 → 종료. **입퇴실/휴가 게시물은 만들지 않습니다.**
  (입퇴실 게시물을 이미 수동으로 만든 날, 중복 생성 없이 집계만 고칠 때 사용)
- 게시 내용은 이미 보관된 `pending_embed`(직전 자정에 확정한 결과)이며, 새로 계산하지 않습니다.
- `morning_post`는 `time=09:00` 스케줄이라 09:00 이후 봇을 켜면 다음 발화가 **이튿날 09:00**이라,
  당일 집계 게시는 자동으로 메워지지 않습니다 — 그 공백을 메우는 용도입니다.

## 영구 상태 파일

| 파일 | 내용 | Git |
|------|------|-----|
| `state.json` | 집계 게시물 ID, 보관 임베드, 일일 게시물 생성 날짜 | **무시(.gitignore)** |
| `sessions.json` | 날짜별·유저별 음성/화면공유 세션(개인정보) | **무시(.gitignore)** |
| `bot.log` | 로그(자동 로테이션 5MB×5) | **무시(.gitignore)** |
| `.env` | 토큰 등 비밀값 | **무시(.gitignore)** |

> 위 파일들은 절대 커밋하지 마세요. 모두 `.gitignore`에 등록돼 있습니다.

`sessions.json` 구조 예시(시각은 Unix epoch 초, 진행 중 세션은 종료 `null`):
```json
{
  "2026-06-15": {
    "123456789": {
      "voice":  [[1718413200, 1718420400], [1718424000, null]],
      "stream": [[1718413260, 1718416800]]
    }
  }
}
```

## 로깅 / 안정성
- 콘솔과 `bot.log`에 동시 출력(`RotatingFileHandler`, 5MB × 5개 로테이션).
- 미처리 예외 traceback도 로그에 기록(`sys.excepthook`), discord 라이브러리 오류도 `bot.log`로 전파.
- 모든 스케줄 루프에 오류 핸들러 + 본문 `try/except` — 개별 실패가 프로세스를 죽이지 않고 다음 주기에 재개됩니다.
- 백그라운드(작업 스케줄러) 실행 시 콘솔이 안 보이므로 **`bot.log`가 유일한 관찰 창구**입니다.

## 배포 — Windows 작업 스케줄러

봇을 부팅 시 자동 실행하고, 비정상 종료 시 재시작하도록 등록합니다.

**준비**
- 파이썬: `D:\python.exe` · 봇 폴더: `C:\Users\백수연\Desktop\discord_bot` · 가상환경 미사용

**등록 절차**
1. 작업 스케줄러 → **작업 만들기**(기본 작업 아님).
2. **일반**: "사용자가 로그온했는지 여부에 관계없이 실행", "가장 높은 권한으로 실행" 체크.
3. **트리거**: 새로 만들기 → **시작할 때**(시스템 시작 시).
4. **동작**: 프로그램 시작
   - 프로그램/스크립트: `D:\python.exe`
   - 인수 추가: `bot.py`
   - 시작 위치: `C:\Users\백수연\Desktop\discord_bot`  ← **반드시 입력**(.env/state.json/sessions.json 경로 때문)
5. **설정**:
   - 작업 실패 시 다시 시작 간격: **1분**, 다시 시작 시도: **999**
   - 이미 실행 중이면: **새 인스턴스를 시작하지 않음**
   - "작업을 요청 시 실행" 체크

**주의**
- 작업 스케줄러의 "실패 시 재시작"은 비정상 종료 코드일 때만 동작합니다. 코드 차원의 안전장치(로깅·예외 처리)가 1차 방어선입니다.
- `.env`가 봇 폴더에 있어야 하고, 시작 위치가 봇 폴더여야 설정이 로드됩니다.
